Анализ целевой аудитории

Провели серию глубинных интервью с потенциальными пользователями (20-30 человек), разделяя их на сегменты: модные энтузиасты, занятые профессионалы, пожилые люди. Использовали инструменты типа Hotjar для анализа поведения на аналогичных сервисах. Результат: персоны пользователей с детализированными сценариями использования.

Конкурентный анализ

Исследовали 15+ существующих решений (Stylebook, Cladwell, Amazon Outfit Builder), составили сравнительную матрицу по 30 параметрам: точность рекомендаций, интеграция с гардеробом, погодная адаптация. Особое внимание уделяли слабым местам конкурентов - например, большинство не учитывает влажность воздуха при подборе. Как итог выделили следующие пункты: Выводы: 80% аналогов не используют погодные данные, нет персонализации под существующий гардероб, упущена интеграция с календарем (подбор под события).

UX-исследование:

70% пользователей хотят видеть альтернативные варианты комплектов

85% предпочитают ручную коррекцию автоматических подборов

Определение ключевого функционала MVP

Основные функции

Разработка MVP для нейрофоторамки, подбирающей одежду под погоду и другие факторы, начинается с определения ключевого функционала, который должен включать базовые возможности для проверки основной гипотезы продукта. Первоочередная задача - реализовать механизм добавления предметов гардероба, позволяющий пользователям загружать фотографии своей одежды через камеру или галерею, с возможностью ручного добавления атрибутов, таких как тип одежды, цветовая гамма и сезонность. Параллельно интегрируются погодные API (OpenWeatherMap или AccuWeather) для получения актуальных данных о температуре, осадках и УФ-индексе, которые будут основными факторами для подбора одежды.

Техническая реализация

На техническом уровне MVP строится на стеке Python (Django/FastAPI) для backend-части с использованием PostgreSQL в качестве базы данных. Для искусственного интеллекта применяются TensorFlow/Keras для компьютерного зрения и анализа изображений одежды, а также Scikit-learn для базовой системы рекомендаций. Фронтенд-часть включает веб-панель на React.js и мобильное приложение на Kotlin/Swift, связанные с backend через REST API и WebSocket для оперативного обновления данных. Архитектура системы представляет собой связку фоторамки с мобильным приложением и веб-интерфейсом, которые взаимодействуют с единым backend, обращающимся к ML-моделям и погодным сервисам.

Финальная демо-версия MVP

Финальная демо-версия MVP включает возможность добавления 10-15 предметов одежды, подбор образов для трех основных погодных сценариев (холодная погода ниже 10°C, теплая погода 15-25°C и дождливые условия), а также базовый интерфейс фоторамки. Алгоритм рекомендаций работает по принципу фильтрации гардероба в зависимости от текущих погодных условий, используя простые правила для начальной версии. После релиза MVP собирается обратная связь для дальнейшего улучшения алгоритма, планируется добавление интеграции с календарем для учета событий при подборе одежды и совершенствование компьютерного зрения для лучшего распознавания тканей и фасонов.
